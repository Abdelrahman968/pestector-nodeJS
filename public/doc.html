Database Schemas

Data models and their relationships in the Pestector API

Description

Stores user account information, authentication details, and subscription status

Fields

Field Type Required Description
profileImage String No URL to user's profile image (default: "/img/user-profile.png")
fullName String No User's full name
username String Yes Unique username (letters, numbers, underscores only)
email String Yes User's email address (unique, validated format)
password String Yes Hashed password
role String Yes User role (user, admin)
location Object No User's location (latitude, longitude, lastUpdated)

Relationships

  • One-to-many with History (user has many history entries)
  • One-to-many with Plant (user has many plants)
  • One-to-many with Reminder (user has many reminders)
  • One-to-many with Notification (user has many notifications)
  • One-to-one with Subscription (user has one active subscription)

Description

Stores classification history for both registered users and guests

Fields

Field Type Required Description
userId ObjectId No* Reference to User (required if guestId not provided)
guestId String No* Guest user ID (required if userId not provided)
imageUrl String Yes URL to the classified image
filename String Yes Original filename
classification Mixed Yes Classification results and details
timestamp Date Yes When the classification was performed

Relationships

  • Many-to-one with User (many history entries belong to one user)
  • One-to-many with ModelFeedback (history entry can have feedback)
